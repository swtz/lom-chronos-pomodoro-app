10. Entendendo como o React renderiza HTML na página (fluxo do React + Vite)

<StrictMode /> | presente no src/main.tsx | é uma boa prática do desenvolvedor
React. É esse componente que 'renderiza' 2x na tela ou que 'imprime' 2x no console
quando existem efeitos colaterais no código. Isso tudo para garantir o funcionamento
correto da aplicação React | É usado somente no modo desenvolvimento (igual o TypeScript).

O React transpila o 'jsx' para o conhecido 'html'. Por isso, no browser, o que se vê
é o 'html' de fato.

Por que SPA (Single Page Application)?
Ora, existe apenas um arquivo 'index.html' em toda a estrutura gerada pelo Vite.
Essa é a única página, de fato, que será carregada ao acessar a aplicação pelo endereço
exibido no console. Por isso, o que o React faz, é permitir o gerenciamento dos seus
componentes criando a "ilusão" de que existem mais páginas na aplicação sendo que,
no fim, todas elas serão renderizadas e colocadas nesse 'index.html';
