Aula: 76. React Router - Iniciando o roteamento da aplicação

O React Router Dom (v7) permite que a aplicação React se comporte como uma
SPA (Single Page Application), pois ele faz o roteamento entre os componentes
que compõe essa aplicação sem a necessidade de recarregar a página (ou a aplicação)
por completo.

Assim como o 'Toastify' o React Router faz uso de Contextos. Assim, o componente
principal dele deve ficar num nível acima dos outros componentes da aplicação
(o componente App, por exemplo).

Estrutura básica:

<BrowserRouter>
  <Routes>
    <Route path='/' element={<Home />} />
    <Route path='/about-pomodoro' element={<AboutPomodoro />} />
    <Route path='*' element={<NotFound />} />
  </Routes>
</BrowserRouter>

Como os elementos Home, AboutPomodoro e NotFound estão dentro do contexto do
componente 'BrowserRouter', é possível usar um componente chamado 'Link'.
Esse componente vai no lugar do elemento JSX 'a'. Assim, o aplicativo React obtém
o comportamento do uma SPA.

<Link
  to='/'
  aria-label='Ir para a Home'
  title='Ir para a Home'
>

A propriedade 'to' é reconhecida pela propriedade 'path' contida no componente 'Route'.
Assim, é garantido o comportamento de uma SPA.
